1.はじめに — なぜ今、個人PCでAIを始めるのか

1.1セミナーの狙い

この社内では、いま 「TeamAgent」 という Slack で動く中央サーバー型の AI エージェントを設計・開発中です。営業16名 + マネージャーが共通して使う、いわば 「会社の脳」 です。フル稼働まで 8.5ヶ月かかります。

一方、本セミナーで扱う 「個人 Claude Code」 は、あなた自身の PC の中で動く、あなた専属の AI です。会社のサーバーは関係なく、今日インストールすれば今日から使えます。守備範囲は「自分の仕事を加速する」こと。

両方とも必要です。会社の脳（TeamAgent）と、個人の右腕（Claude Code）。本セミナーはその「個人の右腕」の入口として、営業のみなさんが明日から使えるレベルまで持ち上げることを目的にしています。

このセミナーで持ち帰ってほしいもの
(1) AI で何が変わるかの感覚 / (2) 個人PCで動く Claude Code のセットアップ手順 / (3) 営業の現場で明日から使える 10 のユースケース / (4) 渡していい情報・渡してはいけない情報の線引き。プログラミング知識は一切不要です。日本語で命令文を書けば動きます。

1.2重要な前提：資料自動化を待たずに価値が出る

TeamAgent の 機能③（提案コンテンツ生成 ＝ 資料自動化） は、本プロジェクトで最も期待されている機能です。ただし、これは Phase 3 で実装されるため、稼働までに 6ヶ月以上かかります。場合によっては、社内事情で計画どおりに進まない可能性もあります。

ここで強調したいのは、「資料自動化が仮に間に合わなくても、個人 Claude Code を使うだけで営業の仕事は本質的に変わる」 ということです。資料が自動で出てくるのは確かに気持ちいい。でも、それより手前にある 「リサーチ・要約・壁打ち・ナレッジ蓄積」 の部分だけでも、個人で十分強力な AI 活用ができます。

本セミナーは、その 「先に手に入る価値」 を示すものです。TeamAgent が完成したら、その上で個人 Claude Code を併用してさらに加速する。今日始めて損はありません。

2.資料自動化以前 — AI 活用の3つの本質的価値

TeamAgent の機能③（資料自動化）を待たなくても、個人 Claude Code を使うだけで 3つの本質的価値 が手に入ります。むしろ、資料自動化はこの3つの先にある「最終形」であって、土台はこの3つです。順番に深掘りします。

2.1ナレッジが個人に蓄積する（→ いずれ会社の脳へ）

AI と対話していくと、不思議なことが起きます。「自分が当たり前にやっている判断ロジック」「自分の営業の勝ちパターン」が、自然と言語化されていくのです。

たとえば「ACME 社の提案、どう組み立てる？」と Claude に聞くと、Claude は「業界・規模・担当者の役職、過去取引……どんな情報があれば答えられる？」と返してきます。それに答えていくうちに、自分の頭の中にあった暗黙のチェックリストが、文字として外に出てきます。

さらに、自分のフォルダ・メール・過去の提案資料を Claude に読ませる経験そのものが、「何が会社の財産になっているか」を可視化する第一歩になります。

💡 TeamAgent への接続
個人 Claude Code で磨いたプロンプト・使い方・ナレッジ整理法は、半年後に TeamAgent が稼働したとき、そのまま 「会社の脳」への入力 として活用できます。個人のスキルが、いずれ組織のインフラに昇格する設計です。

2.2リサーチの劇的効率化

これが営業現場で一番効くやつです。競合動向・業界トレンド・顧客背景の事前リサーチが、従来 1〜2 時間 → 5〜10 分に短縮できます。Web 検索 + 要約 + 論点抽出を、AI が一度にこなすからです。

観点Before（従来の手作業）After（個人 Claude Code）
所要時間60 分7 分
手順Google で 10 個キーワード検索 → 上位記事 5 本を読む → メモする → ChatGPT で要約 → 自分の言葉で再整理Claude Code に「美容業界の最新クチコミ動向を調べて、3つの論点にまとめて」と1行投げる
アウトプットWord に貼り付けたメモ（構造化されてない）3 論点 + 根拠 URL + 営業アングル提案までセット
使える化1回限りそのまま提案書のアウトラインに転用できる

表 2.2事前リサーチの Before / After（例：美容業界の最新クチコミ動向）

重要なのは、「リサーチに費やしていた時間が、本質的な顧客対応・関係構築に回せる」 という点です。AI に頼ることで楽になるのではなく、営業として本当にやりたかった仕事に時間を戻せる のです。

2.3思考の壁打ち相手が常に手元にいる

これは地味ですが、効きます。提案企画の整理・想定問答・トーン調整・社内説明の準備。これまで上司や先輩を巻き込まないとできなかった壁打ちが、AI 相手に何度でもできます。

具体例：

  • メール壁打ち：一度書いた返信案を Claude に「これ受け取った相手はどう感じる？役員視点でも見て」と聞ける。
  • 提案ロジック検証：「この提案の弱点は？」「クライアントから来そうな反論は？」を 3 分で返してくれる。
  • 表現の調整：「もう少しフォーマルに」「業界用語を使った版で」「3行に圧縮して」が即座にできる。
  • 社内説明：「これを上司にどう説明する？1分バージョンと3分バージョンで」とお願いできる。
💡 メンタルモデル
Claude Code は 「24時間そばにいる、忍耐力無限の先輩」 だと思ってください。「こんなこと聞いていいのかな」と遠慮する必要はありません。むしろ、最初の3案を AI に出させてから、人間の先輩に「3案のどれが一番マシ？」と聞くほうが、先輩の時間も節約できます。

4.セミナーで実演するハンズオン

ここからは、セミナー当日に その場で見せる / その場でやってもらう パートです。座学だけだと「ふーん」で終わるので、手を動かす設計にしてあります。

4.1セットアップの全体像

Claude Code を個人 PC で動かすには、4 ステップ・約 5 分で完了します。詳細手順書は別途配布する 「Claude Code セットアップ手順書 v1」（PDF）を参照してください。ここでは全体像だけ示します。

Step 1（1分）Claude Pro（個人）または Max（プロ向け）プランを契約する。会社契約か個人立替かは社内ルール次第。
Step 2（2分）Claude Code をインストール。CLI 版（ターミナル）かデスクトップアプリ版を選ぶ。営業の方はデスクトップアプリ推奨。
Step 3（1分）初回起動 → ブラウザで Anthropic アカウント認証 → 完了。
Step 4（1分）最初のプロンプト「このフォルダの中身を要約して」を打ってみる。動いたら成功。

4.2デモプロンプト5本（実演用）

講師が実演します。それぞれ 「プロンプト例」「期待される出力」「ポイント」 の3点セットで示します。

Prompt 1 — 過去資料からの強みパターン抽出

プロンプト例：
このフォルダの過去提案資料を読んで、共通している強みパターンを3つ挙げて。
それぞれに「どの資料で言っていたか」も添えて。
期待出力3つの強みパターン + 根拠ファイル名 + 推奨される強調ポイント
ポイント自分の暗黙知が文字に出てくる。新人教育にも転用可能。

Prompt 2 — Web リサーチ + 提案アングル

プロンプト例：
ACME 社（仮）の最新動向を Web で調べて、当社の提案アングルを3つ提案して。
各アングルは「根拠 → 切り口 → 想定リアクション」の順で。
期待出力3アングル + 各アングルの根拠 URL + 商談での切り出し方
ポイント事前リサーチが 60分 → 7分に。商談前日の夜にこれを 1回投げる習慣を。

Prompt 3 — 議事録から ToDo 抽出

プロンプト例：
この議事録（添付）から、次アクションと担当者を Markdown のチェックリストにして。
期限が議事録内に書かれていなければ「期限要確認」とマークして。
期待出力- [ ] アクション内容 @担当者（期限）のチェックリスト
ポイントそのまま Slack に貼れる。捏造防止のため「書かれていない情報は要確認」と明示するのがコツ。

Prompt 4 — メール返信案の客観レビュー

プロンプト例：
このメール返信案、相手の役員視点で読んだときに違和感があるか指摘して。
「言い回しが強すぎる」「責任の所在が曖昧」など具体的に。
期待出力違和感ポイントのリスト + 修正案 + 強さレベル評価
ポイント"24時間いる先輩"の真骨頂。送信前 30秒の保険として使う。

Prompt 5 — Excel 数値から優先順位提案

プロンプト例：
この Excel（添付）の数値から、来月の見込み案件の優先順位を提案して。
判断基準も明示して。私が同意できるかチェックできるように。
期待出力優先順位ランキング + 判断基準（金額・確度・タイミング等）+ 各案件への一言コメント
ポイント判断基準を明示させることで、AI 任せにせず人間が最終判断できる。

5.個人PCでできることの境界線

「Claude Code は万能なの？」という質問への答えは 「個人の範囲では万能に近い、組織の範囲では無理」 です。3色で分けます。

🟢 / 🟡 / 🔴 の境界線一覧

難度 項目 理由 担当
🟢 余裕 自分のローカルファイル分析（過去資料・Excel・PDF） 自分の PC に元々あるデータ。権限不要。 個人 Claude Code
🟢 余裕 自分の Gmail / Drive 連携 自分のアカウント範囲で OAuth 認可するだけ。 個人 Claude Code
🟢 余裕 Web リサーチ（公開情報） 誰でもアクセスできる情報を AI が要約・抽出。 個人 Claude Code
🟢 余裕 文書ドラフト作成（メール・提案書骨子・社内連絡） テキスト生成は AI の最も得意とする領域。 個人 Claude Code
🟢 余裕 議事録要約・チェックリスト化 長文 → 構造化の典型タスク。 個人 Claude Code
🟢 余裕 個人レベルの自動化スクリプト Claude にコードを書かせて自分の PC で回す範囲なら自由。 個人 Claude Code
🟡 工夫 個人での動画分析（yt-dlp + Claude Code） 1本ずつ手動で取得して投入。重い。会社全体の蓄積には不向き。 個人 Claude Code（自分用）
🟡 工夫 個人用 MCP 接続（Slack / Notion / カレンダー） 自分のアカウント範囲のみ。設定にやや慣れが要る。 個人 Claude Code
🟡 工夫 自分専用のナレッジベース構築 自分のフォルダを整理して Claude に検索させる。基盤づくりに時間がかかる。 個人 Claude Code
🔴 不可 他人のメール / Slack を横断検索 個人 PC には他人のアクセス権限がない。会社側のサーバーで一括認可する必要。 → TeamAgent
🔴 不可 朝8時の自動 push 配信 24時間稼働するサーバーが必要。個人 PC は閉じる/寝るので無理。 → TeamAgent
🔴 不可 マネージャー横串の進捗集約 16人分のデータを統合する権限・基盤が必要。 → TeamAgent
🔴 不可 会社全員が共有するナレッジDB 個人 PC は共有先になれない。中央サーバー + pgvector が必要。 → TeamAgent
🔴 不可 24時間稼働の自動応答 常時起動するインフラが必要。 → TeamAgent

結論
🟢🟡 まではあなたの仕事を確実に変える。🔴 は TeamAgent ができたら補う。 個人 Claude Code を今日始めても、半年後に TeamAgent と衝突しません。むしろ、その時に両方使えるあなたが組織で一番強くなります。

6.セキュリティとガバナンス

営業のみなさんが一番不安なのはここだと思います。「クライアント情報を AI に渡していいの？」。3つの観点から答えます。

6.1渡していい情報・渡してはいけない情報

分類具体例渡してよいか
🟢 渡していい 公開情報（プレスリリース・公式サイト・業界ニュース） 自由に渡してよい
🟢 渡していい 自分の業務メモ・自分が書いた提案ドラフト 自由に渡してよい
🟢 渡していい 自分が普段アクセスできる範囲のクライアント情報（議事録・自分宛メール） 原則 OK。ただし機密度の高い案件は社内ルール確認
🔴 渡してはいけない 個人情報（マイナンバー・口座番号・パスポート番号・健康情報） 絶対に渡さない
🔴 渡してはいけない NDA で「外部 AI 利用禁止」と明記されているクライアント資料 NDA 違反になるので渡さない
🔴 渡してはいけない 契約上明確に AI 使用が禁じられている情報 渡さない。契約文面を確認
🟡 迷ったら 機密度高めの社内戦略・財務情報・人事情報 情シス / コンプラに確認してから

表 6.1AI に渡してよい情報・ダメな情報の早見表

⚠ 判断に迷ったら
「これ渡していいかな？」と一瞬でも思ったら、渡さずに情シス / コンプライアンス担当に確認してください。社内で確認した実績が積み上がれば、AI 活用のルールも整備されていきます。最初の数件の確認は、後続の人を助けます。

6.2Anthropic / Claude のデータ利用方針

Anthropic（Claude を作っている会社）の公式方針として、API 経由・Claude Code 経由のチャットは原則として学習に使われません（オプトイン式：明示的に許可した場合のみ）。これは個人ユーザーにとって安心材料です。

ただし、規約は変わる可能性があります。本セミナー資料の記述だけを根拠にせず、最新の Anthropic 公式ポリシー（Usage Policies / Data Policy）を必ず確認してください。営業時に「弊社はこの規約のもとで AI を使っている」と説明できることが重要です。

確認先Anthropic 公式サイト → Privacy / Usage Policies のページ
確認のタイミング導入時 + 半年ごとの定期チェック推奨
営業時の説明「AI ベンダー側で学習に使われない設定で運用しています」と一言添える

6.3社内のクレデンシャル管理

業務利用のレベルになると、API キーやアカウント情報の管理を個人で抱え込まない方がよいです。

  • サブスク（Pro / Max プラン）：会社払いか個人立替か、社内ルールに従う。経費精算で立て替える場合は、情シスと会計担当に事前確認。
  • API キー：業務利用なら必ず IT 部門経由で発行・管理。個人の Gmail に届くキーをそのまま使い続けるのは避ける。
  • 退職時の引き継ぎ：個人アカウントで業務データを抱えると、退職時にトラブルになる。最初から「会社の管理下」を意識する。
💡 推奨フロー
Step 1 で Pro / Max プランを契約する際、「これが業務利用になりそう」と感じたら最初から IT 部門と会話する。後から法人契約に切り替えるのは手間が増えます。

7.今日から始める3ステップ

全部覚える必要はありません。この3ステップだけ持ち帰ってください。残りはやりながら覚えればいい話です。

1Claude Pro / Max プランを契約する

  • 個人用なら Pro 月 ¥3,000 程度、本格利用なら Max 月 ¥30,000 程度。
  • 会社払いか個人立替かは社内ルール確認。経費申請の際は「営業力強化のための AI 活用」と申請目的を明記。
  • 最初の月は Pro で十分。使い倒してから Max に上げるかどうか判断する。

2Claude Code をインストールする

  • Anthropic 公式ドキュメント（docs.anthropic.com/claude-code）の手順に沿ってインストール。
  • 営業の方はまず デスクトップアプリ版を推奨。GUI で動くので CLI に抵抗があっても問題なし。
  • CLI 版を試したい方は、npm install -g @anthropic-ai/claude-code でインストールできます。
  • つまずいたら、本セミナーの後に配布する 「Claude Code セットアップ手順書」 PDF を参照。

3今週の案件1つで「事前リサーチ」を投げる

  • 今週商談予定のクライアント名 + 業界キーワードを Claude Code に渡し、「最新動向と提案アングルを3つ」 と頼んでみる。所要 5 分。
  • 翌日、別のユースケース（議事録要約 or メール壁打ち）を1回試す。
  • 1週間続けたら、自分にとってどのユースケースが一番効くかが分かる。そこを軸に習慣化。
🎯 ゴール設定
「2週間後の自分が、Claude Code なしの仕事に戻れなくなっている」状態を目指してください。そうなれば、半年後に TeamAgent が出てきたとき、すんなり接続できます。

8.FAQ — よくある不安に答える

営業の方からよく聞かれる質問を 6 つに絞って先回りで答えます。それでも残る疑問はセミナーの最後の Q&A 時間で。

Q1. AI に頼り切ると、自分の力が落ちないか？
道具は道具、判断は人間です。むしろリサーチ・要約・トーン調整を AI に任せた時間で、本質的な顧客対応に集中できます。AI が出した3案から1つを選ぶ・修正するという作業は、白紙から書くより判断力が鍛えられる側面もあります。「自分で書く力」より「自分で選ぶ力」が今後の営業の差別化です。
Q2. クライアント情報を AI に渡すのは怖い
正しい不安です。第6章の表に沿って、公開情報 + 自分の業務メモから始めましょう。慣れてきたら、自分宛のメールや議事録に範囲を広げる。マイナンバー・口座番号・NDA 明記の機密情報は 絶対に渡さない。迷ったら情シス。これだけ守っていれば事故は起きません。
Q3. プロンプトのコツが分からない
最初は 自然な日本語の命令文でまったく問題ありません。「○○について調べて、3つにまとめて」「このメール、相手の立場で違和感あるか見て」程度で十分動きます。セミナー後に 「営業向け 使い回せるプロンプト集 v1」 を配布予定です。「丁寧に頼む」「箇条書きで欲しいと指示する」「具体例を1つ添える」、この3つを意識すればまずは OK。
Q4. TeamAgent ができたら、Claude Code はいらなくなる？
なりません。両方使います。住み分けは第1章の表のとおりです。TeamAgent は「会社の脳」、Claude Code は「個人の右腕」。例えば、TeamAgent が朝8時に届けた営業進捗サマリーを見て、その後の個別案件深掘りは Claude Code、というのが理想的なフロー。両方使える人が組織で一番強くなる設計です。
Q5. コストは？
個人 Pro プランは 月 ¥3,000 程度、本格利用の Max プランは 月 ¥30,000 程度です（2026年5月時点・為替変動あり）。会社が払うか個人が払うかは社内ルール次第。営業1人が月に 10時間でも工数削減できれば、Pro プランは確実に元が取れます。最初は Pro から始めて、必要なら Max に上げる流れが安全です。
Q6. マネージャーに見えるところで使っていいか？
むしろ推奨します。こっそり使うものではありません。「AI 活用で営業力を強化している取り組み」として、マネージャー・経営層に見せていく方が、組織全体の AI 投資判断にも貢献します。商談前に「Claude Code でリサーチしました」と言える状態がカッコいい時代です。
セミナー後のサポート
Slack の #ai-help チャンネル（仮）で質問を受け付けます。プロンプトに困ったら気軽に投げてください。本セミナーの講師 + 早期に使い始めた営業メンバーがフォローします。
